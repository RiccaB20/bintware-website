# Knowledge Base SEO — Bintware

Documenti di riferimento per scelte SEO sul sito Bintware.com.
Vengono caricati selettivamente quando una richiesta riguarda SEO,
contenuti del blog, struttura URL, sitemap, meta tag, titoli, immagini
ottimizzate o redirect.

## Indice

- [01-principi-generali.md](01-principi-generali.md) — principi base derivati dalla guida Google Search Central: crawler, struttura URL, tag HTML, immagini, anti-pattern e falsi miti.
- [02-documentazione_seo.md](02-documentazione_seo.md) — linee guida architetturali per sviluppatori: rendering JS/DOM, navigazione e link, HTML semantico, controllo scansione/indicizzazione, dati strutturati JSON-LD.
- [03-architettura-seo-anti-spam.md](03-architettura-seo-anti-spam.md) — direttive anti-spam e di conformità: divieto cloaking, gestione paywall, link spam (`nofollow`/`sponsored`), redirect ingannevoli, doorway, testo nascosto, keyword stuffing, sicurezza UGC, reputazione del sito.
- [04-seo-javascript-dinamico.md](04-seo-javascript-dinamico.md) — regole per SPA / contenuti caricati dinamicamente: API History vs frammenti, ciclo di vita Googlebot/WRS, cache busting / fingerprinting, lazy loading e infinite scroll, soft 404, web components, JSON-LD dinamico.
- [05-architettura-seo-link-navigazione.md](05-architettura-seo-link-navigazione.md) — architettura link e navigazione: markup `<a href>`, anchor text descrittivo (no "Fai clic qui"), `alt` per img-link, qualificatori `nofollow`/`sponsored`/`ugc`, link interni con riferimenti incrociati.
- [06-architettura-seo-sitemap.md](06-architettura-seo-sitemap.md) — sitemap XML: limiti (50K URL / 50 MB), URL assoluti e canonici, image sitemap (`xmlns:image`), `<lastmod>` accurato; `<priority>` e `<changefreq>` sono inefficaci, ignorati da Google.
